从0到1，搭建Hexo博客

从0到1，教你在Windows上搭建Hexo博客

准备工作

1.安装 Git Bash

  • 下载地址 GitBash
  • 安装步骤 一路next
  • 安装完成，打开Git Bash，输入git version查看版本

2.安装 Node JS

Hexo是基于Node JS的静态博客 ，后面安装Hexo的插件要用到npm命令。

  • 下载地址(LTS版本) Node JS
  • 安装步骤：点击msi文件，在Custom Setup这个界面勾选Add to PATH
  • 安装完成，打开Git Bash，输入node -v查看版本

3.安装 Hexo

以上是搭建Hexo的基础，下面开始创建一个属于自己的hexo博客

  • 先创建一个文件，用于存放自己的blog文件，在该文件夹下，右键打开Git Bash
  • 安装hexo命令：npm i -g hexo
  • 安装完成，查看hexo版本，输入hexo -v
  • 初始化命令 hexo init，等待片刻之后，在blog文件夹下可以看到以下文件
  • 各个文件夹的作用：
    • node_modules: 依赖包
    • public：存放生成的页面(要发布项目才有)
    • scaffolds：命令生成文章的模版
    • source：用命令创建的各种文章
    • themes：下载的主题
    • _config.yml：整个博客的配置信息
    • db.json：source解析所得到的
    • package.json：项目所需模块项目的配置信息

本地调试Hexo博客

  1. 在blog文件夹目录下，右键打开Git Bash，输入hexo s
  2. 在本地浏览器打开localhost:4000查看博客
  3. hexo相关命令
    • hexo s：启动本地服务器，用于预览
      • hexo s是hexo server的缩写
      • 预览同时可以对文章、主题修改，刷新页面即可
      • 对hexo根目录的_config.yml修改，需要重启服务器
    • hexo clean：清除缓存文件
      • 重启时需要清除缓存
    • hexo d：生成静态网站，部署到github仓库
      • hexo d是hexo deploy的缩写
    • hexo g：生成静态文件到public文件夹
      • hexo g是hexo generate的缩写
      • 便于查看网站生成的静态文件或手动部署网站
    • hexo new "My First Blog"：新建一篇文章
      • 生成的文章在根目录\source_posts里面
    • hexo new page aboutme：新建一个标题为aboutme的页面
      • 页面不会出现在文章列表和归档中，也不支持设置分类和标签

总结

个人的本地Hexo博客就搭建完成了，后面会介绍如何发布到Github上面，之后我们就可以在互联网上通过浏览器访问到自己的博客。
